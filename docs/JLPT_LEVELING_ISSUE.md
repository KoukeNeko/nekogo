# 待辦：JLPT 分級「N1 出現簡單詞」的觀察與對策

> 狀態：**已診斷，暫不處理**（分級本身正確）。記錄以免遺忘。

## 觀察
使用者在 N1 牌組看到「很簡單的單字」。

## 診斷結論：分級是對的、忠於來源，不是 bug
- **核心正確**：已知簡單詞都正確在 N5（会う/食べる/水/犬/学校/見る/私/本/時間/友達…）。「最簡單級別優先」（`Math.max` of matched levels）運作正常。
- **N1 共 2,681 詞**：
  - **86%（2,295）含漢字、確為進階詞**：圧倒・斡旋・暗殺・圧迫・明白・暗示・安静…合理。
  - **14%（386）純假名**：擬態語/口語詞（あっさり・きっぱり・じっくり・でかい・だったら・お袋）。看起來簡單（短假名）但屬進階/口語 register，**確實來自 tanos `n1.csv`**（且只在 n1）。

## 為何「看起來簡單卻在 N1」
- **擬態語/擬音語**（あっさり、ずるずる、てっきり）對學習者其實難，JLPT 列高級。
- **口語/俗語**（でかい、だったら、お袋）不在初級教科書 → 歸 N1。

## 先天限制（非 pipeline 錯）
2010 年後 JLPT **官方不再公布單字表**；tanos 是 2010 前的估計清單，少數分級（でかい、だったら）以現代眼光可議。這是資料源限制。

## 對策（未來做）
真正問題在「**牌組內順序**」——純假名擬態語剛好排前面，造成「N1 很簡單」的錯覺。建議用**詞頻排序**：
1. `vocab` 加 `freq_rank`（來源：TUBELEX-JA BSD 或 jpstats CC0，授權乾淨）。
2. 新卡 `ORDER BY freq_rank`（先學高頻，像 jpdb）。
3. 可選：標記/降權「高頻卻被分到 N1」的可疑詞。

實作面：一條 ETL（建 freq 表）+ `build-content-db.mjs` 寫入 `vocab.freq_rank` + `cardRepository.getDueCards` 改 `ORDER BY`。
